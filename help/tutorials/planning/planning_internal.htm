Planning Tutorial - Internal

This version of the Planning Tutorial discusses planning using Incremental Epic's in-built planning functionality.

Overview

Incremental Epic automatically uses a project's tasks to create a plan. The plan is available on a seperate 'Plan' tab on the project editor:

The system will automatically add new and emergent tasks to the plan, and update their status. The system will allow you to change the plan dates of started tasks but not completed or cancelled tasks.

Pre-requisites

To generate or update a plan the following must be true:

  • you must have the planning plug-in(s) installed;
  • you must have the read-write planning activity;
  • the planning preferences must be set;
  • the project must have a status of 'Awaiting Work' or later;
  • there must be tasks in the project.

Set Up

  1. Open your user preferences and select 'System Settings / Planning' (you must have the administration activity to do this);
  2. Choose 'Planning at customer task level'. The yard task option may be disabled, dependent on your version of Epic.

  3. Note that you could also set up external planning simultaneouly - but it is not recommended.

     

View the Plan

  1. Open a project at the 'Awaiting Work' stage or later, which has tasks (either imported from an estimate or manually added as emergent work);
  2. The 'Plan' tab will be available and will show all tasks as plan items with a default duration of two days:

  3. Much like the parent project's editor, the plan has a read-only and read-write mode. Note that this is entirely seperate to the parent editor's mode.
    The plan opens read-only. Right-clicking opens a menu that allow you to zoom in or out, and to choose whether to display the task's duration in elapsed days.

Print the Plan

  1. In either read-only or read-write mode you can print the plan;
  2. This done via the two print icons in the top right. You choose between printing the plan on a single sheet of A4, or on multiple sheets for better clarity;
  3. In either case clicking a print icon will open the Print Preview screen which allows further customisation of the print - page layout, boundaries etc:

Update the Plan

Clicking the 'Edit' icon in the top right (much like opening a read-write editor) will switch the pla to read-write mode. This will only happen if no-one else has already done this. You can switch back to read-write by clicking the 'Save' icon that replaces the 'Edit' icon. Note that whilst the plan is open read-write no-one else can edit tasks so the plan should be in read-write mode for a short a time as possible.

In read-write mode you can reschedule tasks and change their duration. You can do this by double-clicking on the task to open a dialog in which you can change the task dates. Or you can use the drag'n'drop functionality of the plan.

Hovering over any non-complete task will show you a 'move' icon. When this is displayed you can hold down the left mouse button to drag the task left or right and change its dates.

If instead you hover over the left or right edges of the task you will see a 'resize' icon . Holding down the left mouse button will allow you to drag that edge whilst leaving the other unmoved - i.e. changing the duration of the task.

Dependencies

You can also add dependencies - task 'B' cannot start until task 'A' is complete. To do this:

  1. Click to select task 'A' - its border will become dashed to indicate it has been selected;
  2. Hold down the SHIFT button and click task 'B';
  3. The dependency will be set up and indicated via a black filled arrow; task 'B' will move to start after task 'A';
  4. To remove the dependency repeat exactly the same sequence of actions on the two tasks.

A note on dependencies: dependencies are a way of automatically setting the start date of a task based on the end date of another task. However as soon as you explicitly set the start date of a task (using drag'n'drop or the pop-up dialog) then any dependencies it has are ignored. Ths system assumes that as soon as you have manually set a date, automatic dates should be ignored.
